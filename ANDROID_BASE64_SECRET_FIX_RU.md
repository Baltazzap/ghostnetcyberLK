# Исправление `base64: invalid input`

## Причина

GitHub Secrets были добавлены через Windows PowerShell с передачей значения
в стандартный ввод `gh`. PowerShell мог добавить к Base64 скрытый символ
перевода строки `CRLF`.

Обычная команда Linux:

```bash
base64 --decode
```

считала этот символ недопустимым и останавливала Android-сборку.

## Что исправлено

Android workflow теперь:

1. получает секрет;
2. удаляет CRLF, пробелы, BOM и другие символы, не относящиеся к Base64;
3. декодирует значение через Python с `validate=True`;
4. проверяет минимальный размер JKS;
5. открывает JKS через `keytool`;
6. проверяет пароль и alias до запуска Gradle.

Также исправлены PowerShell-скрипты настройки GitHub Secrets. Теперь они
передают значение в `gh` без завершающего перевода строки.

Существующие GitHub Secrets можно не удалять: обновлённый workflow
нормализует уже сохранённое значение.
