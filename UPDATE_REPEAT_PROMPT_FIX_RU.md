# Исправление повторного предложения обновления

## Причина

Раньше каждый Android-релиз загружался под одним именем:

```text
GhostNet-Cyber-VPN.apk
```

а Windows-релиз:

```text
GhostNet-Cyber-VPN-Setup.exe
```

После замены файла браузер, CDN или хостинг мог вернуть старую копию из
кэша. При этом `version.json` уже содержал новую версию. Пользователь
устанавливал старый файл, и приложение снова видело, что серверная версия
выше установленной.

Такое же поведение возникает, если на сайт случайно загрузить новый
`version.json`, но оставить старый APK или EXE.

## Исправление

Каждый релиз теперь получает уникальное имя:

```text
GhostNet-Cyber-VPN-1.0.6-7.apk
GhostNet-Cyber-VPN-Setup-1.0.6-7.exe
```

Следующий релиз будет иметь другие имена, например:

```text
GhostNet-Cyber-VPN-1.0.7-8.apk
GhostNet-Cyber-VPN-Setup-1.0.7-8.exe
```

Поэтому старый файл из кэша больше не может подменить новый релиз.

`version.json` теперь содержит отдельные платформенные блоки `android` и
`windows`, но сохраняет старые поля для совместимости со старыми версиями
приложения.

## Правильный порядок загрузки на сайт

Для версии `1.0.6+7` сначала загрузить:

```text
/downloads/GhostNet-Cyber-VPN-1.0.6-7.apk
/downloads/GhostNet-Cyber-VPN-Setup-1.0.6-7.exe
```

Проверить, что оба файла скачиваются.

Только после этого загрузить:

```text
/downloads/version.json
```

Старые APK и EXE можно оставить на сайте: новые версии используют другие
имена и не конфликтуют с ними.
