# Windows: исправление второго окна после оплаты

## Проблема

После успешной или отменённой оплаты браузер открывал ссылку:

```text
ghostnet://payment-result
```

Windows повторно запускал `GhostNetCyberVPN.exe`, из-за чего появлялось второе окно приложения.

## Что исправлено

В Windows runner добавлена single-instance защита через Win32:

- создаётся именованный Mutex;
- первый процесс продолжает обычный запуск;
- второй процесс обнаруживает уже запущенное приложение;
- существующее окно восстанавливается и выводится на передний план;
- второй процесс сразу завершается;
- второе Flutter-окно не создаётся.

Приложение уже проверяет статус платежа через API каждые 3 секунды, поэтому результат оплаты продолжит отображаться в первом окне.

## Поведение

- Приложение открыто: браузер возвращает пользователя в уже открытое окно.
- Приложение свёрнуто: окно восстанавливается.
- Приложение закрыто: запускается одно окно, затем проверяется сохранённый платёж.
